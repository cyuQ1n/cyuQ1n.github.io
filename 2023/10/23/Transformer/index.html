Transformer

论文链接：https://arxiv.org/pdf/1706.03762.pdf

代码仓库：https://github.com/tensorflow/tensor2tensor

Recurrent Neural Networks（RNN）、Long Short-term Memory（LSTM）和 Gated Recurrent Neural networks（GRU）等神经网络在序列模型、机器翻译等领域取得了广泛的应用。但由于 Recurrent Models 的时序关系，导致无法在训练过程中进行并行化，因此这类模型的训练时间较长，且效率不高。 即使通过 factorization tricks 和 conditional computation 等进行改进，但缺陷仍然存在。

文章提出了新的 Transformer 模型，丢弃了循环神经网络的结构，而完全依赖于 Attention 机制来获取输入与输出的全局依赖关系。

Model Architecture

Transformer 遵循 Encoder-Decoder[1]的整体架构，对两个部分都使用堆叠的自注意力机制和全连接层，整体架构如下图所示。

Encoder and Decoder Stacks

  • Encoder：编码器由6个相同层堆叠在一起的堆栈形成，每个层包含两个子层，其中一个是多头自注意力机制，另外一个是简单的、按位置的全连接前馈神经网络（Feed Forward）。在每个子层后面，都进行了一个残差连接（Resnet）和层归一化（Layer Normalization）。即：每个子层的输出是LayerNorm(x+Sublayer(x))LayerNorm(x + Sublayer(x))LayerNorm(x+Sublayer(x))，其中 Sublayer(x)Sublayer(x)Sublayer(x) 是子层本身实现的函数。为实现残差连接，模型中的所有层（包括嵌入层 Embedding layers）产生的输出维度均为 dmodel=512d_{model}=512dmodel​=512
  • Decoder：解码器也是由6个相同堆叠在一起的堆栈形成，除了每个编码器层中的两个子层之外，解码器插入第三个子层，该子层对编码器堆栈的输出执行多头注意（即图2对应的 Decoder 部分的中间子层）。与编码器类似，在每个子层周围使用残差连接，然后进行层归一化。除此之外，还修改了解码器堆栈中的自注意子层，以防止位置关注后续位置（Masked Muti-Head Attention）。这种掩蔽，再加上输出嵌入被偏移一个位置的事实[2]，确保了位置iii的预测只能依赖于位置小于iii的已知输出。

Attention

Attention 机制可以描述为将查询（query）和一组键值对（a set of key-value pairs ）映射到输出，其中query、key、value 和输出都是向量。输出是作为V的加权和计算的，其中分配给每个 value 的权重是由 query 与相应 key 进行复杂函数计算得到的。

Scaled Dot-Product Attention

如图 3 所示，其中左图被称为 “Scaled Dot-Product Attention”，其输入包括dkd_{k}dk​维的 query 和 key ，dvd_{v}dv​维的 value。计算步骤如下：

  1. 计算query与key的点积(Dot Product)

  2. 点积结果除以dk\sqrt { d _ { k } }dk​​

  3. 对前一步结果利用 softmax 函数获得V的权重

在实践中，会同时计算一组 query 的注意力函数，这样的一组 query 会打包成矩阵QQQ[3]。key 和 value 也会被打包成矩阵KKK和VVV。注意力机制计算式如下：

attention(q,k,v)=softmax(QKTdk)Vattention(q,k,v) =softmax(\frac{QK^{T}}{\sqrt{d_{k}}})V attention(q,k,v)=softmax(dk​​QKT​)V

两种最常用的注意力函数是加性注意力[4]和点积注意力。点积注意力和我们的算法是一样的，除了比例因子是1dk\frac{1}{\sqrt{d_{k}}}dk​​1​。加性注意力使用一个具有单个隐藏层的前馈网络来计算兼容性函数。两者在理论复杂性上相似，但在实践中，点积注意力更快，更节省空间，因为它可以使用高度优化的矩阵乘法代码来实现。

为什么在注意力机制计算值要乘以1dk\frac{1}{\sqrt{d_{k}}}dk​​1​？

当dkd_kdk​值较小时，两种机制的表现相似，当dkd_kdk​值较大时，加性注意优于点积注意。作者怀疑，对于较大的dkd_kdk​值，点积的大小会变大[5]，从而将softmax函数推入具有极小梯度的区域。为了抵消这个影响，将点积乘以1dk\frac{1}{\sqrt{d_{k}}}dk​​1​。

Applications of Attention in our Model

Transformer 以三种不同的方式应用 Multi-head attention：

  • 在“encoder-decoder attention”层中，querys来自前一个解码器层，而keys和values来自编码器的输出。这允许解码器中的每个位置都参与输入序列中的所有位置。这模仿了sequence-to-sequence模型[7]中典型的编码器-解码器注意机制。
  • 编码器包含自关注层。在自关注层中，所有的keys、values和querys都来自同一个地方。在这种情况下，是编码器中前一层的输出。编码器当前层中的每个位置都可以处理编码器前一层中的所有位置信息。
  • 解码器中的自注意层允许解码器中的每个位置注意到解码器中的所有位置直至并包括该位置。为防止解码器中的向左信息流以保持自回归特性，通过屏蔽[8]softmax输入中对应于非法连接<span class=“hint–top hint–rounded” aria-label=““illegal connections”（非法连接）通常指的是在特定任务或应用中，某些位置之间的连接是不合理的或不允许的。这可能是因为在某些NLP任务中，生成的单词不能依赖于未来的单词，或者在图像处理中，像素之间的连接必须遵循一定的规则。本文在为了确保这些非法连接不会影响注意力计算，作者将这些连接所对应的数值设置为负无穷大，这样在进行 softmax 计算时，它们的概率会接近于零，从而有效地忽略了这些非法连接。”>[9]的所有值来实现缩放点积注意力。

为什么是Self-Attention？

作者主要从计算复杂度、可并行化计算量[15]以及网络中长距离依赖的路径长度[16]三个方面，对自注意力层（self-attention layers）与常用于将一个变长序列的符号表示（x1,...,xn）（x_1,...,x_n）（x1​,...,xn​）映射到另一个等长度序列（z1,...,zn）（z_1, ..., z_n）（z1​,...,zn​）的递归（recurrent）层和卷积（convolutional）层进行比较。

  1. 总计算复杂度： 作者比较了各种层的计算复杂度。自注意力层在每一层中将所有位置连接起来，并执行了O(1)O(1)O(1)的顺序操作。递归层则需要进行O(n)O(n)O(n)个顺序操作。此外，当序列长度nnn小于表示维数ddd时，自注意层比循环层更快，这是机器翻译中最先进模型使用的句子表示最常见的情况。为了提高涉及很长序列的任务的计算性能，可以将自注意机制限制为只考虑以输出位置为中心的输入序列中大小为rrr的邻域。这会增加最大值路径长度为O(n/r)O (n/r)O(n/r)。
  2. 可并行计算量： 作者考察了各种层的可并行计算量，即可以同时进行的最小操作数量。自注意力层在这方面表现出色，因为它可以并行地执行操作，而递归层则需要按顺序执行。
  3. 长距离依赖的路径长度： 作者比较了不同类型层构成的网络中任意两个输入和输出位置之间的最大路径长度。具有核宽度k<nk < nk<n的单个卷积层不能连接所有输入和输出位置对。要实现这一点，需要使用O(n/k)O(n/k)O(n/k)个连续核的卷积层或O(logk(n))O(log_k(n))O(logk​(n))的膨胀卷积（dilated convolutions）层，从而增加了网络中任意两个位置之间的最长路径长度。此外，卷积层通常比递归层更昂贵，计算复杂度增加了kkk倍。但是，分离卷积（separable convolutions）可以显著减少计算复杂度，降低到O(k⋅n⋅d+n⋅d2)O(k·n·d+n·d^2)O(k⋅n⋅d+n⋅d2)。不过，即使在k=nk = nk=n的情况下，分离卷积的计算复杂度仍然等于自注意力层和逐点前馈层（point-wise feed-forward layer）的组合（这是作者模型中采用的方法）。

此外，self-attention可以产生更多可解释的模型。作者从Transformer模型中检查注意力分布，并在文章附录中给出和讨论示例，发现：不仅个体注意力头清楚地学会执行不同的任务，许多注意力头似乎表现出与句子的句法和语义结构相关的行为。这意味着自注意力机制可以帮助模型更好地理解文本中不同部分之间的关系，从而提高了模型的可解释性。

Training & Result

Training

Training Data and Batching

模型在标准的 WMT 2014 English-German 数据集[17]和更大的 WMT 2014 English-French 数据集[18]进行训练，句子对按近似序列长度进行批处理。每个training batch包含一组句子对，其中包含大约25000个源tokens和25000个目标tokens。

Hardware and Schedule

模型在一台带有8个NVIDIA P100 GPU的机器上训练。对于使用本文中描述的超参数的基本模型，每个训练步骤大约需要0.4秒。基本模型进行了总共10万步，大约12小时的训练。对于我们的大型模型，步长为1.0秒。训练了30万步，约3.5天。

  1. Residual Dropout：将Dropout应用于每个子层的输出，然后将其添加到子层输入并归一化。将dropout应用于编码器和解码器堆栈中的嵌入和位置编码之和。对于基本模型，Pdrop=0.1P_{drop} = 0.1Pdrop​=0.1。

Result

Machine Translation

在 WMT 2014 English-German 翻译任务中，大型Transformer模型（表中的transformer (big)）比之前报道的最佳模型(包括集成模型)高出2.0 BLEU2.0 \ BLEU2.0 BLEU以上，建立了新的最好BLEU分数28.4。训练时间为3.5天，使用的是8个P100 GPU。甚至Transformer基本模型也超过了所有以前发表的模型和集合，而训练成本只是其他任何竞争模型的一小部分。

在WMT 2014 English-to-French 翻译任务上，Transforner（big）获得了41.0的BLEU分数，优于之前发布的所有单一模型，而训练成本不到之前最先进模型的1/4。针对这一任务，Pdrop=0.1P_{drop} = 0.1Pdrop​=0.1，而不是0.3。

对于基本模型，使用通过平均最后5个 checkpoints ，获得的单个模型，这些 checkpoints 每隔10分钟写入一次。对于大型模型，取最后20个 checkpoints 的平均值。作者使用束搜索[22]，束大小为4，长度惩罚α = 0.6。这些超参数是在开发集上实验后选择的。将推理过程中的最大输出长度设置为输入长度+ 50，但尽可能早地终止。
